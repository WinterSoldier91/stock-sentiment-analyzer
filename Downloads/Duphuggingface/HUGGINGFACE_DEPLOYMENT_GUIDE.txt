✅ CORRECT: There is NO standalone "Streamlit" SDK option!

❌ WRONG: Files pushed to subdirectories like "Downloads/Duphuggingface/"
✅ CORRECT: Files MUST be in the root directory of the Space
✅ SOLUTION: Clone the HF Space fresh, copy files to root, then push

MISTAKE #3: Wrong Port Configuration
❌ WRONG: Using port 8501 (Streamlit default)
✅ CORRECT: Hugging Face Spaces uses port 7860
✅ MUST SET: In Dockerfile: --server.port=7860
✅ MUST SET: In README.md: app_port: 7860

MISTAKE #4: Incorrect README.md Configuration
❌ WRONG: sdk: streamlit (when using Docker)
❌ WRONG: Including extra fields like license, sdk_version
❌ WRONG: Missing app_port field
✅ CORRECT: Minimal YAML with sdk: docker and app_port: 7860

MISTAKE #5: Database Path Not Configured for Persistent Storage
❌ WRONG: Hardcoded DB_NAME = 'sentiment_history.db'
✅ CORRECT: Use conditional path for /data directory

MISTAKE #6: Hugging Face Persistent Storage is NOT Free
❌ WRONG: Assuming HF's built-in persistent storage is free
❌ WRONG: Telling users HF persistent storage is free
✅ CORRECT: HF persistent storage is a PAID feature
✅ SOLUTION: Use Supabase PostgreSQL (100% free, 500MB)

MISTAKE #7: Missing database.py in Dockerfile
❌ WRONG: Dockerfile only copies app.py and future_article_fixer.py
❌ WRONG: Forgetting to copy database.py module
✅ CORRECT: Must also copy database.py
✅ FIX: Add `COPY database.py .` to Dockerfile

MISTAKE #8: Database Connection During Docker Build
❌ WRONG: Calling `init_database()` at module level in app.py
❌ WRONG: Database functions that crash when connection fails
❌ WRONG: Environment variables not available during Docker build
✅ CORRECT: Remove module-level database initialization
✅ CORRECT: Add graceful fallback in database connection functions
✅ FIX: Return None from get_db_connection() on error, check in all functions

MISTAKE #9: Missing psycopg2-binary in requirements.txt
❌ WRONG: Only having SQLite dependencies
❌ WRONG: Missing PostgreSQL driver
✅ CORRECT: Add `psycopg2-binary>=2.9.0` to requirements.txt

STEP 3.3: Create database.py Module
------------------------------------
Create database.py with PostgreSQL connection functions:

```python
import os
import psycopg2
from psycopg2.extras import RealDictCursor
from datetime import datetime, timedelta
import pytz
import pandas as pd

def get_db_connection():
    """Get PostgreSQL connection with graceful error handling"""
    try:
        return psycopg2.connect(
            host=os.getenv('DB_HOST'),
            database=os.getenv('DB_NAME'),
            user=os.getenv('DB_USER'),
            password=os.getenv('DB_PASSWORD'),
            port=os.getenv('DB_PORT', 5432)
        )
    except psycopg2.Error as e:
        print(f"Database connection error: {e}")
        return None
    except Exception as e:
        print(f"Unexpected database error: {e}")
        return None

def init_database():
    """Initialize database tables (already done in Supabase)"""
    pass

def get_historical_data(ticker, days=30):
    """Get historical sentiment data from PostgreSQL"""
    conn = get_db_connection()
    if conn is None:
        return pd.DataFrame()
    
    try:
        with conn.cursor(cursor_factory=RealDictCursor) as cursor:
            cursor.execute("""
                SELECT * FROM sentiment_history 
                WHERE ticker = %s 
                ORDER BY timestamp DESC 
                LIMIT %s
            """, (ticker, days))
            rows = cursor.fetchall()
            return pd.DataFrame(rows)
    except Exception as e:
        print(f"Error fetching historical data: {e}")
        return pd.DataFrame()
    finally:
        conn.close()

def can_fetch_new_data(ticker):
    """Check if we can fetch new data (15-minute cache)"""
    conn = get_db_connection()
    if conn is None:
        return True, "Database not available - fetching fresh data"
    
    try:
        with conn.cursor() as cursor:
            cursor.execute("""
                SELECT last_fetch FROM cache_metadata 
                WHERE ticker = %s
            """, (ticker,))
            result = cursor.fetchone()
            
            if not result:
                return True, "No cache found"
            
            last_fetch = result[0]
            now = datetime.now(pytz.UTC)
            time_diff = now - last_fetch
            
            if time_diff.total_seconds() > 900:  # 15 minutes
                return True, "Cache expired"
            else:
                return False, f"Cache valid for {int(900 - time_diff.total_seconds())} more seconds"
    except Exception as e:
        print(f"Error checking cache: {e}")
        return True, "Error checking cache - fetching fresh data"
    finally:
        conn.close()

def update_cache_metadata(ticker, article_count):
    """Update cache metadata"""
    conn = get_db_connection()
    if conn is None:
        return
    
    try:
        with conn.cursor() as cursor:
            cursor.execute("""
                INSERT INTO cache_metadata (ticker, last_fetch, article_count)
                VALUES (%s, %s, %s)
                ON CONFLICT (ticker) 
                DO UPDATE SET 
                    last_fetch = EXCLUDED.last_fetch,
                    article_count = EXCLUDED.article_count
            """, (ticker, datetime.now(pytz.UTC), article_count))
            conn.commit()
    except Exception as e:
        print(f"Error updating cache metadata: {e}")
    finally:
        conn.close()

def store_sentiment_data(ticker, parsed_and_scored_news):
    """Store sentiment data in PostgreSQL"""
    conn = get_db_connection()
    if conn is None:
        return
    
    try:
        with conn.cursor() as cursor:
            for article in parsed_and_scored_news:
                cursor.execute("""
                    INSERT INTO sentiment_history 
                    (ticker, headline, sentiment_score, timestamp, url)
                    VALUES (%s, %s, %s, %s, %s)
                    ON CONFLICT (ticker, headline, timestamp) DO NOTHING
                """, (
                    ticker,
                    article['headline'],
                    article['sentiment_score'],
                    article['timestamp'],
                    article['url']
                ))
            conn.commit()
    except Exception as e:
        print(f"Error storing sentiment data: {e}")
    finally:
        conn.close()
```

------------------------------------------------------------
CAUSE: Wrong port configuration

FIX:
1. Dockerfile must have: --server.port=7860
2. README.md must have: app_port: 7860
3. Both must match!

----------------------------------------------

1. Verify files are in root directory
2. Push your actual app.py file
3. Restart the Space

ERROR: "Build error" with database.py missing
---------------------------------------------
CAUSE: Dockerfile doesn't copy database.py

FIX: Add `COPY database.py .` to Dockerfile

ERROR: "Database connection error" during build
-----------------------------------------------
CAUSE: App tries to connect to database during Docker build
CAUSE: Environment variables not available during build

FIX:
1. Remove `init_database()` call from module level in app.py
2. Add graceful fallback in get_db_connection() function
3. Check for `conn is None` in all database functions

ERROR: "ModuleNotFoundError: No module named 'psycopg2'"
-------------------------------------------------------
CAUSE: Missing psycopg2-binary in requirements.txt

FIX: Add `psycopg2-binary>=2.9.0` to requirements.txt

ERROR: App works but data doesn't persist
------------------------------------------
CAUSE: Database credentials not added to HF Spaces secrets

FIX: Add all 5 database environment variables to HF Spaces secrets:
- DB_HOST
- DB_NAME
- DB_USER
- DB_PASSWORD
- DB_PORT

ERROR: Database resets on restart
----------------------------------
CAUSE: Using SQLite instead of PostgreSQL
CAUSE: Database credentials not properly configured

FIX:
1. Use Supabase PostgreSQL (not SQLite)
2. Verify all database secrets are added to HF Spaces
3. Check database.py has proper error handling
